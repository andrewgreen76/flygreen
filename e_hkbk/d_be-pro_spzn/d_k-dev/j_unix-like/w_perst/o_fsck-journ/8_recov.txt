==================================================================================================================
RECOVERY METHODS : 
==================================================================================================================

When a crash or a power outage occurs in the middle of writing to FS , the FS is stuck in corrupt (inconsistent) state.
  |
  |
 (a) use fsck to tend to the entire disk
  |
 (b) pre-emptively log in a JOURNAL (and then fix) : 
      . write-ops on the data blocks 
      . target inodes
      . target bitmap / location
      . target data blocks 



==================================================================================================================
FS STABILITY : 
==================================================================================================================

Stable state : 
 . a file system state where there is consistency between the bitmaps , inodes , and data blocks
 . must be regained/maintained before and after changes to FS (even if it means dropping the file changes) 



==================================================================================================================
JOURNALING DURING RECOVERY : 
==================================================================================================================

Ways to regain stability :
  |
 (a) roll back (revert) the changes to FS , achieving the initial stable state 
 (b) replay (re-enact) all the changes to FS , achieving the final stable state 
  |
  . We will need JOURNALING.
